# Public Launch Checklist - 2026-07-11

## Before Publishing The Demo Page

- Replace `hello@example.com` in `archimedes_demo_share_page.html`.
- Decide the public URL.
- Confirm screenshots load from the deployed path.
- Confirm example reports open from the deployed path.
- Add the synthetic-data disclaimer.
- Check mobile layout manually.
- Remove any private/local file paths from public copy.

## Before First Outreach

- Pick 5 leads from `first_batch_outreach_2026-07-11.csv`.
- Verify the contact route manually.
- Open the demo page link in an incognito browser.
- Add the campaign row to the tracker.
- Decide where replies should go.
- Prepare the follow-up but do not send it yet.

## Before First Social Post

- Choose one screenshot.
- Use one proof point.
- Use one operating rule.
- Include that the example is synthetic/non-medical if needed.
- Avoid claiming customer outcomes.
- Track the post in the content tracker.

## During The First Week

- Send 5-10 careful manual touches.
- Publish 2-3 posts.
- Capture exact reply language.
- Track every objection.
- Do not change the whole strategy after one weak signal.
- Do not buy paid tools yet.

## End Of Week Review

- Which segment replied?
- Which message got interest?
- Did anyone ask to see an example?
- Did anyone describe the pain in their own words?
- Is "messy exports -> action reports" landing?
- Should the next demo be ecommerce, SaaS, or another specific workflow?
