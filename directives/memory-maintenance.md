# Memory Maintenance

Tags: #directive #memory

Use during private/main sessions or heartbeat maintenance. Do not expose private memory content in shared group chats.

## Steps

1. Review recent `memory/YYYY-MM-DD.md` files.
2. Identify concrete decisions, preferences, lessons, commitments, and durable project context.
3. Add only high-signal items to [[MEMORY]].
4. Remove or revise outdated long-term entries when they are no longer useful.
5. Add links from structured notes in [[second-brain/README|Second Brain]] when a topic deserves its own page.

## Keep

- Durable preferences
- Decisions that will affect future work
- Project state that would be costly to rediscover
- Mistakes or lessons that prevent repeat errors

## Skip

- Secrets
- One-off chatter
- Raw transcripts
- Temporary implementation details
- Anything unsafe to surface outside a private context
