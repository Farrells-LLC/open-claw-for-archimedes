# OpenClaw Memory System

Tags: #memory #openclaw #obsidian

OpenClaw memory works best when important context is written to files and then rediscovered later through startup context, targeted search, or Obsidian graph navigation.

## Core Files

- [[MEMORY]] - curated long-term memory
- [[memory/]] - daily raw notes
- [[HEARTBEAT]] - proactive maintenance checklist
- [[AGENTS]] - workspace behavior rules
- [[SOUL]] - agent identity and tone
- [[USER]] - human context
- [[TOOLS]] - local tool notes

## Operating Pattern

1. Write important events, decisions, and lessons into daily notes.
2. Periodically distill stable items into [[MEMORY]].
3. Link related notes with Obsidian wiki-links.
4. Keep long-term memory short, current, and high signal.
5. Store repeatable procedures in [[directives/README|directives]] instead of burying them in chat.

## Privacy Boundary

Long-term memory can contain personal context. It should be loaded and summarized only in private/main sessions, not in shared group channels.
